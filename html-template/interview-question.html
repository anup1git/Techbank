Collections

1. HashMap Internal Working:

HashMap uses a hash table to store key-value pairs. It internally uses an array to store the elements, where each element is a linked list (bucket) containing key-value pairs with the same hash code.

2. Difference:

  • Array vs ArrayList: Arrays have a fixed size, while ArrayLists can dynamically grow.
  • HashMap vs TreeMap: HashMap is unordered and uses hashing, while TreeMap is ordered and uses a red-black tree.
  • Map vs HashMap: Map is an interface, and HashMap is a class implementing that interface.
  • ArrayList vs LinkedList: ArrayList uses a dynamic array, while LinkedList uses a doubly linked list.

3. Concurrency and ConcurrentHashMap:

Concurrency is the ability to run multiple threads concurrently. ConcurrentHashMap is a thread-safe implementation of the Map interface, designed for concurrent use without the need for external synchronization.

4. Fail-Safe Iterator vs Fail-Fast Iterator:

Fail-Safe Iterator makes a copy of the collection and iterates over the copy, so it does not throw ConcurrentModificationException if the collection is modified during iteration. Fail-Fast Iterator, on the other hand, throws ConcurrentModificationException if the collection is modified during iteration.

OOPs Concepts

SOLID Principles:

  • Single Responsibility Principle (SRP): A class should have only one reason to change.
  • Open/Closed Principle (OCP): Software entities should be open for extension but closed for modification.
  • Interface Segregation Principle (ISP): Clients should not be forced to depend on interfaces they do not use.
  • Liskov Substitution Principle (LSP): Objects of a superclass should be replaceable with objects of its subclasses without affecting the functionality.
  • Dependency Injection (DI): Objects should depend on abstractions, not on concretions.

Design Patterns

  • Singleton Pattern: Ensures a class has only one instance and provides a global point of access to it. Example: Spring Framework's singleton beans.
  • Factory Pattern: Defines an interface for creating objects, but lets subclasses decide which class to instantiate. Example: java.util.Calendar class uses getInstance() method to create Calendar objects.
  • Abstract Factory Pattern: Provides an interface to create families of related or dependent objects without specifying their concrete classes. Example: javax.xml.parsers.DocumentBuilderFactory.
  • Prototype Pattern: Used when creating an instance of a class is expensive or complex. It involves creating new objects by copying an existing object, known as the prototype. Example: java.lang.Object's clone() method.

How to Create a Singleton Class?

To create a singleton class, you can use the following approach:

        public class Singleton {
            private static final Singleton instance = new Singleton();
        
            private Singleton() {}
        
            public static Singleton getInstance() {
                return instance;
            }
        }

How to Create an Immutable Class?

To create an immutable class, follow these steps:

  • Make the class final so it cannot be extended.
  • Make all fields private and final.
  • Do not provide setter methods for fields.
  • Ensure that methods do not modify the state of the object.
  • Return a new instance for methods that modify the state, instead of modifying the current instance.
