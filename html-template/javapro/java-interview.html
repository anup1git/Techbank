Java Interview Questions

Q: What are the key features of Java 8?

Answer:

Java 8 introduced several new features, including:

  • Lambda expressions
  • Stream API
  • Default methods in interfaces
  • Functional interfaces
  • New Date and Time API (java.time package)

Q: Explain the internal working of HashMap in Java.

Answer:

HashMap in Java uses a hash table to store key-value pairs. It uses an array to store the elements, where each element is a linked list (bucket) containing key-value pairs with the same hash code. When a key-value pair is inserted into the HashMap, the hash code of the key is used to determine the index of the bucket in the array. If multiple key-value pairs have the same hash code, they are stored in the same bucket as a linked list.

Q: What is the difference between an array and an ArrayList in Java?

Answer:

The main differences between an array and an ArrayList in Java are:

  • Arrays have a fixed size, while ArrayLists can dynamically grow.
  • Arrays can contain primitive types and objects, while ArrayLists can only contain objects.
  • Arrays use square brackets ([]) for declaration, while ArrayLists use the ArrayList class.
  • Arrays are more efficient for direct element access, while ArrayLists are more flexible and provide additional methods for manipulation.

Q: Concurrency and ConcurrentHashMap

Answer:

Concurrency is the ability to run multiple threads concurrently. ConcurrentHashMap is a thread-safe implementation of the Map interface, designed for concurrent use without the need for external synchronization.

Q: Fail-Safe Iterator vs Fail-Fast Iterator

Answer:

Fail-Safe Iterator makes a copy of the collection and iterates over the copy, so it does not throw ConcurrentModificationException if the collection is modified during iteration. Fail-Fast Iterator, on the other hand, throws ConcurrentModificationException if the collection is modified during iteration.

Q: Explain SOLID Principles in Java

Answer:

Principle Description
Single Responsibility Principle (SRP) A class should have only one reason to change.
Open/Closed Principle (OCP) Software entities should be open for extension but closed for modification.
Interface Segregation Principle (ISP) Clients should not be forced to depend on interfaces they do not use.
Liskov Substitution Principle (LSP) Objects of a superclass should be replaceable with objects of its subclasses without affecting the functionality.
Dependency Injection (DI) Objects should depend on abstractions, not on concretions.

Q: Name few Design Patterns

Answer:

Pattern Description Example
Singleton Pattern Ensures a class has only one instance and provides a global point of access to it. Spring Framework's singleton beans.
Factory Pattern Defines an interface for creating objects, but lets subclasses decide which class to instantiate. java.util.Calendar class uses getInstance() method to create Calendar objects.
Abstract Factory Pattern Provides an interface to create families of related or dependent objects without specifying their concrete classes. javax.xml.parsers.DocumentBuilderFactory.
Prototype Pattern Used when creating an instance of a class is expensive or complex. It involves creating new objects by copying an existing object, known as the prototype. java.lang.Object's clone() method.

Q: How to Create a Singleton Class?

Answer:

To create a singleton class, you can use the following approach:

            public class Singleton {
                private static final Singleton instance = new Singleton();
            
                private Singleton() {}
            
                public static Singleton getInstance() {
                    return instance;
                }
            }

Q: How to Create an Immutable Class?

Answer:

To create an immutable class, follow these steps:

  • Make the class final so it cannot be extended.
  • Make all fields private and final.
  • Do not provide setter methods for fields.
  • Ensure that methods do not modify the state of the object.
  • Return a new instance for methods that modify the state, instead of modifying the current instance.
